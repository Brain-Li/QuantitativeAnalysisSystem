# 状态记忆与重置规则

说明应用中 **数据集选择、字段配置、时间范围、股票选择、字段条件筛选、趋势分析所选指标** 在「切换模块」「刷新页面」「退出登录」以及「股票列表 · 重置」下的行为。实现要点：`src/app/sessionPersistence.ts`（退出登录时清理的存储键）、`QuantitativeApp.tsx`（退出逻辑）、`DataTable.tsx`（股票列表重置）、`StockTrendAnalysis.tsx`（趋势分析与股票/指标对齐逻辑）。

---

## 1）选择数据集

| 场景 | 行为 |
|------|------|
| 切换模块 | **记忆**（保留上次选择） |
| 刷新页面 | **记忆** |
| 退出重登 | **重置**：恢复默认，**默认勾选当前全部数据集**（存储中清空选中，数据加载完成后由逻辑全选） |
| 股票列表 · 重置 | **不控制**数据集选择 |

**产品逻辑**：数据集是强业务筛选，用户连续分析时希望保持一致；**退出登录表示会话结束**，恢复初始状态，避免下次登录仍带着上次的子集选择导致错乱。

**实现说明**：选中项持久化在 `qas_selected_dataset_ids_v1`。刷新时 React 状态先从存储恢复；数据集列表异步加载完成后，若存储中的 id 仍存在于当前列表则**保持该子集勾选**（与「切换模块」一致）；若存储为空（例如退出登录后写入的空数组）或条目均已失效则**默认勾选当前全部数据集**。

---

## 2）字段配置（列表显示哪些列、列顺序）

| 场景 | 行为 |
|------|------|
| 切换模块 | **记忆** |
| 刷新页面 | **记忆** |
| 退出重登 | **记忆**（本地持久化 `datatable_field_configs_v1`，退出登录**不删除**） |
| 股票列表 · 重置 | **不控制**字段配置 |

**产品逻辑**：字段配置是**视图偏好**，属于个人习惯，应长期保留。

---

## 3）时间筛选（开始日期～结束日期）

| 场景 | 行为 |
|------|------|
| 切换模块 | **记忆** |
| 刷新页面 | **记忆** |
| 退出重登 | **重置为默认**：股票列表 **近 3 天**；趋势分析 **近 7 天**（删除对应 localStorage 项后按各模块默认初始化） |
| 股票列表 · 重置 | 股票列表时间恢复为 **近 3 天** |

**产品逻辑**：时间筛选是高频分析条件，同一会话内保持连续体验；退出后恢复默认，避免下次打开仍是超大时间范围导致加载卡顿。

---

## 4）股票筛选（选中个股 / 全选）

| 场景 | 行为 |
|------|------|
| 切换模块 | **记忆** |
| 刷新页面 | **记忆** |
| 退出重登 | **股票列表**：清空持久化，恢复为**全选**语义；**趋势分析**：清空并回到**默认「选择股票」**（无选中，由模块内逻辑与数据对齐） |
| 股票列表 · 重置 | 清空当前选择，恢复 **全选**（与列表内「未选或选满全部代码」一致） |

**产品逻辑**：股票筛选偏单次分析行为，会话内保持连续；**退出后必须清空**持久化，避免下次登录默认带大量个股导致卡顿。

**实现说明**：股票列表使用 `datatable_selected_stocks_v3`；趋势分析的股票选择在 `trend_analysis_state_v1` 内。两处在**股票代码列表尚未加载完成**（`stockOptions` 为空）时**不会把选中清空**，待选项就绪后再与当前可选代码取交集，从而切换模块、刷新页面后仍能恢复上次选择。

---

## 5）股票列表 · 字段条件筛选

| 场景 | 行为 |
|------|------|
| 切换模块 | **记忆** |
| 刷新页面 | **记忆** |
| 退出重登 | **全部清空** |
| 股票列表 · 重置 | **全部清空**（条件与逻辑恢复为默认「所有」） |

**产品逻辑**：与业务筛选、时间、股票并列，可随「重置」与退出一起恢复。

---

## 6）趋势分析 · 所选指标（主图与对比图）

| 场景 | 行为 |
|------|------|
| 切换模块 | **记忆**（保留上次所选指标；对比图内各图的指标一并随 `trend_analysis_state_v1` 恢复） |
| 刷新页面 | **记忆** |
| 退出重登 | **重置**：随趋势分析整块状态删除（`trend_analysis_state_v1` 被清空），回到**无选中指标**等与初始态一致的表现 |
| 股票列表 · 重置 | **不控制**趋势分析指标 |

**产品逻辑**：所选指标属于分析条件，与趋势分析内时间范围、股票、图表类型等同一套持久化；同一会话内切换模块或刷新应保持连续体验；退出表示会话结束，与其它趋势分析状态一并清理。

**实现说明**：主图 `selectedMetrics` 与对比图内每条 `chart.selectedMetrics` 均写入 **`trend_analysis_state_v1`**。模块内会用「当前样本行上可识别的**数值型字段列表**」与已选指标取交集，以适配当前数据集。关键点在于：**在数值型指标列表尚未就绪**（尚无样本行、列表暂时为空）时，**不得**把已选指标清空并写回存储——否则会在切换模块、首屏拉数等阶段误覆盖持久化，表现为「记忆丢失」。待样本与字段就绪后，再执行交集，去掉当前数据中已不存在的字段名。

---

## 本地存储键（摘要）

| 键名 | 含义 | 退出登录时 |
|------|------|------------|
| `qas_selected_dataset_ids_v1` | 选中数据集 id | 写入 `[]`（随后加载为全选） |
| `datatable_field_configs_v1` | 字段配置 | **保留** |
| `datatable_date_range_v3` | 股票列表日期 | **删除** |
| `datatable_selected_stocks_v3` | 股票列表选中代码 | **删除** |
| `datatable_field_filters_v1` | 字段条件筛选 | **删除** |
| `trend_analysis_state_v1` | 趋势分析 UI 状态（含日期、股票、**所选指标**、图表类型、对比图列表等） | **删除** |
| `trend_analysis_date_range_v1` | 旧版趋势日期（兼容） | **删除** |

---

## 股票列表 ·「重置」按钮

一键将 **时间（近 3 天）**、**股票（全选）**、**字段条件筛选（清空）** 恢复为上述默认；**不修改**右上角数据集选择与字段配置面板。
